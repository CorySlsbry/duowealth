# DuoWealth — App Store Submission Guide

This repo is a production-ready Progressive Web App. It can be shipped to BOTH
the Microsoft Store and the Google Play Store with the assets already in this
repository.

**Live URL:** https://duowealth.vercel.app

---

## 1. Microsoft Store (direct PWA submission)

The Microsoft Store accepts installable PWAs directly — no MSIX packaging
required. Microsoft uses [PWABuilder](https://www.pwabuilder.com/) to generate
the MSIX.

1. Deploy the app to production (`vercel --prod`) and confirm the PWA installs
   from Edge or Chrome on desktop.
2. Go to https://www.pwabuilder.com/ and enter `https://duowealth.vercel.app`.
3. PWABuilder will score your manifest, service worker, and icons. Every field
   in `public/manifest.json` is already filled in.
4. Click **Package for Stores → Windows**. Download the generated MSIX.
5. Upload the MSIX to Microsoft Partner Center:
   https://partner.microsoft.com/en-us/dashboard/windows/overview
6. Fill in the store listing (screenshots, description, age rating).
7. Submit for review — typical turnaround is 24-48 hours.

## 2. Google Play Store (Trusted Web Activity via Bubblewrap)

Google Play requires a native Android wrapper. Google provides
[Bubblewrap](https://github.com/GoogleChromeLabs/bubblewrap) which turns a
PWA into a Trusted Web Activity (TWA) APK/AAB.

### One-time setup

```bash
npm install -g @bubblewrap/cli
bubblewrap doctor        # install JDK 17 + Android SDK if missing
```

### Build the AAB

```bash
cd duowealth
bubblewrap init --manifest https://duowealth.vercel.app/manifest.json
# Accept the defaults, then:
bubblewrap build
```

The pre-filled `twa-manifest.json` in this repo matches the PWA manifest so
the identity / theme / icon match the installed app.

### Add Digital Asset Links

Bubblewrap will print a SHA-256 fingerprint. You MUST host the
`assetlinks.json` file Bubblewrap generates at:

`https://duowealth.vercel.app/.well-known/assetlinks.json`

(Drop it into `public/.well-known/assetlinks.json` and redeploy.)

### Submit the AAB

1. Sign in to Google Play Console: https://play.google.com/console
2. Create a new app, fill in the store listing.
3. Upload the `.aab` file from `./app-release-bundle.aab`.
4. Complete the content rating, data safety, and target audience questionnaires.
5. Submit for review — typical turnaround is 1-7 days for the first review.

## 3. Before you submit (checklist)

- [ ] Privacy Policy published at `https://duowealth.vercel.app/privacy`
- [ ] Terms of Service at `https://duowealth.vercel.app/terms`
- [ ] At least 3 screenshots (1080 x 1920 portrait) in `public/store/`
- [ ] Feature graphic (1024 x 500) in `public/store/feature.png`
- [ ] App listing description (short = 80 chars, full = 4000 chars)
- [ ] Content rating completed in both Microsoft Partner Center and Play Console
